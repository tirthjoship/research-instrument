# Archive — completed specs & plans

**These are historical. They describe work that is DONE and shipped. Do NOT pick one
up as a task** — that is how drift happens. They are kept only for retrieval of a past
design detail.

- Current state + next action: `docs/STATUS.md` (single source of truth).
- Decisions: `docs/adr/` (an ADR captures the durable decision a spec/plan implemented).
- Phase history: `docs/PHASE_LOG.md`.

If you need an *active* spec/plan, it lives in `docs/superpowers/specs/` or
`docs/superpowers/plans/` — and if those dirs are empty, there is no open build for the
current phase. Start the next one with `superpowers:brainstorming` → `writing-plans`.
